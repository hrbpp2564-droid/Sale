วิธีที่ 1 แก้ไข “ข้อมูล” — ไม่ต้อง deploy เลย

นี่คือสิ่งที่คุณจะทำบ่อยที่สุด เช่น เพิ่มยอดขายเดือนใหม่ หรือแก้ตัวเลข — แค่อัปเดตใน Supabase แล้วเว็บเปลี่ยน อัตโนมัติทันที ที่ผู้ใช้กดรีเฟรช ไม่ต้องแตะ GitHub เลย

  • เข้า Supabase → Table Editor → ตาราง dashboard_data
  • แก้คอลัมน์ payload (แถว id = bwp) — แก้ตัวเลขที่ต้องการ แล้ว Save
  • เสร็จ! เปิดเว็บ hrbwp02-beep.github.io/Sale/ รีเฟรช → เห็นข้อมูลใหม่ทันที
ผมเตรียม “เครื่องมืออัปเดตข้อมูล” แบบกรอกฟอร์มให้ด้วย (ดูแท็บ BWP-Data-Editor) — กรอกตัวเลขเดือนใหม่ กดปุ่ม ส่งเข้า Supabase ได้เลย ไม่ต้องแก้ JSON เอง

วิธีที่ 2 แก้ไข “หน้าตา/โค้ด” — deploy อัตโนมัติ

ถ้าแก้ดีไซน์ เพิ่มกราฟ เปลี่ยนโลโก้ ฯลฯ (แก้ไฟล์ index.html) — ผมตั้ง GitHub Actions ให้แล้ว ทุกครั้งที่ push ขึ้น main เว็บจะ build + เผยแพร่ใหม่อัตโนมัติใน ~1 นาที

  • อัปโหลดไฟล์ .github/workflows/deploy.yml (การ์ดดาวน์โหลดด้านล่าง) เข้า repo
  • ไป Settings → Pages → Build and deployment → Source เปลี่ยนเป็น GitHub Actions
  • ต่อจากนี้ commit/แก้ไฟล์ใด ๆ → ดูสถานะ deploy ได้ที่แท็บ Actions (✓ เขียว = ขึ้นแล้ว)

ตัวอย่าง: เพิ่มยอดขายเดือน มิ.ย. 2569

ใน Supabase แก้ payload → ใส่ตัวเลขเดือนที่ 6 ลงในอาเรย์ (ตำแหน่งที่ null อยู่):

// payload.valueByYear["2569"] — มูลค่า (ล้านบาท)
"valueByYear": {
  "2569": [32.5, 32.4, 38.4, 53.4, 59.4, 62.0, null, ...]
  //                                    ↑ ใส่ มิ.ย. ตรงนี้
}
โครงสร้าง payload เหมือนกับ window.VDATA ในไฟล์ — มี KPIS, valueByYear, volumeByYear, CUSTOMERS, PRODUCTS, forecast ฯลฯ แก้ส่วนไหนเว็บก็เปลี่ยนตามส่วนนั้น
