1. บทบาทของแอดมิน

แอดมินคือ ผู้ดูแลระบบ ที่ เข้าดูข้อมูลได้ทุกหน้า และแก้ไข/อัปเดตข้อมูลได้ รวมถึงจัดการข้อมูลลูกค้า ยอดขายรายเดือน และค่า KPI ต่าง ๆ คู่มือนี้ครอบคลุมทั้งการอ่านค่าและการจัดการข้อมูลทั้งหมด

5. การอัปเดตข้อมูล (Data Editor)

เครื่องมือแก้ไขข้อมูล: https://sale-hrbpp2564-droids-projects.vercel.app/bwp-data-editor

แก้ตัวเลขแล้วเว็บจริงเปลี่ยนทันทีที่รีเฟรช โดยไม่ต้อง deploy ใหม่

ขั้นตอน

  1. เปิดหน้า Data Editor
  2. แก้ไขค่าในตาราง:
    • KPI หลัก — label, ค่า, หน่วย ของการ์ดด้านบน
    • ยอดขายรายเดือน 2569 — เติมเดือนใหม่ในช่องว่าง (มูลค่า / ปริมาณ)
  3. กด 💾 บันทึกเข้า Supabase
  4. กลับมาหน้าแดชบอร์ด กด ↻ โหลดใหม่ หรือรีเฟรช (Ctrl+Shift+R)
ปุ่ม “บันทึก” จะกดได้เมื่อมีการแก้ไขแล้วเท่านั้น · ข้อมูลเก็บที่ Supabase (cloud) ทุกคนเห็นค่าใหม่ทันที
หน่วยข้อมูลสำคัญ: ช่องปริมาณกรอกเป็น พัน Kg (ระบบจะคูณ 1,000 ให้เองเวลาแสดงผลเต็มจำนวน) ช่องมูลค่ากรอกเป็น บาท — ตรวจหน่วยให้ถูกก่อนบันทึกทุกครั้ง

6. การจัดการ PIN ผู้บริหาร

PIN สำหรับโหมดผู้บริหารปัจจุบันคือ bwp@2545 เก็บเป็นค่า hash ที่ปลอดภัยในฐานข้อมูล Supabase (ตาราง app_secrets คีย์ dashboard_pass)

หากต้องการเปลี่ยน PIN ให้แจ้งทีมพัฒนา หรืออัปเดตค่า hash ในตาราง app_secrets ผ่าน Supabase โดยตรง — ไม่ควรเก็บ PIN เป็นข้อความธรรมดาในไฟล์โค้ด

อย่าเผยแพร่ PIN ผู้บริหารในที่สาธารณะ และเปลี่ยนเป็นระยะเพื่อความปลอดภัย

7. แหล่งข้อมูล & การ Deploy

  • ข้อมูลสด: ดึงจาก Supabase ทุกครั้งที่เปิดเว็บ (ตาราง dashboard_data)
  • ข้อมูลสำรอง: ถ้าต่อ Supabase ไม่ได้ ใช้ข้อมูลฝังในไฟล์ (data.js) อัตโนมัติ
  • Deploy: push โค้ดขึ้น branch main → Vercel build + เผยแพร่อัตโนมัติ ~1 นาที
  • หากต้องล้าง cache ผู้ใช้ ปรับเลขเวอร์ชันใน index.html (พารามิเตอร์ v=)
