# BWP Vantage — คู่มือตั้งค่า GitHub Pages + Supabase

แดชบอร์ดนี้ออกแบบเป็น **static web app + ฐานข้อมูล** — โฮสต์ฟรีถาวรได้ และต่อ
ฐานข้อมูลจริงได้ คู่มือนี้มี 2 ส่วน ทำตามทีละขั้น

---

## ส่วน A — โฮสต์ถาวรฟรีด้วย GitHub Pages

> ผลลัพธ์: ลิงก์ถาวรเช่น `https://<username>.github.io/bwp-sales/` ส่งให้ทั้งบริษัทได้

**สิ่งที่คุณต้องทำ:**
1. เปิดเมนู **Import** ในช่องแชต → กด **Connect GitHub** → อนุญาตสิทธิ์
2. ส่งข้อความกลับมาบอกผม

**สิ่งที่ผมทำให้ (อัตโนมัติหลังคุณเชื่อม GitHub):**
- สร้าง repo + push ไฟล์แดชบอร์ดทั้งหมด
- เปิด GitHub Pages ให้ (branch `main`, โฟลเดอร์ root)
- ส่งลิงก์ถาวรให้คุณ

**อัปเดตข้อมูลภายหลัง:** ส่งไฟล์ Excel เดือนใหม่ → ผมแปลง + push → ลิงก์เดิมอัปเดตทันที

---

## ส่วน B — ฐานข้อมูลจริงด้วย Supabase (Postgres ฟรีขั้นต้น)

> ผลลัพธ์: ข้อมูลยอดขายอยู่ใน Postgres จริง แก้ผ่านหน้าเว็บได้ แดชบอร์ดดึงสด

**สิ่งที่คุณต้องทำ:**
1. สมัคร [supabase.com](https://supabase.com) (ฟรี) → สร้าง project ใหม่ (เลือก region Singapore ใกล้ไทยสุด)
2. ไปที่ **SQL Editor** → วางเนื้อหาไฟล์ `backend/supabase-schema.sql` → กด **Run**
   (สร้างตาราง customers / products / sales + views + สิทธิ์อ่านสาธารณะ)
3. ไปที่ **Project Settings → API** → คัดลอก 2 ค่านี้ส่งให้ผม:
   - **Project URL** (เช่น `https://abcdxyz.supabase.co`)
   - **anon public key** (ขึ้นต้น `eyJ...` — เป็น key อ่านอย่างเดียว ปลอดภัยที่จะแปะในหน้าเว็บ)

   > ⚠️ **อย่าส่ง `service_role` key** — อันนั้นเป็นสิทธิ์เขียน ห้ามแปะในหน้าเว็บ

**สิ่งที่ผมทำให้ (หลังได้ URL + anon key):**
- เขียนสคริปต์ import แปลงข้อมูลจาก `ยอดขาย 69.xlsx` → push เข้า Supabase
- แก้แดชบอร์ดให้ดึงข้อมูลจาก Supabase แบบเรียลไทม์ (แทน `data.js`)
- มี fallback ใช้ `data.js` หากต่อ DB ไม่ได้ (กันล่ม)

---

## สรุปสถาปัตยกรรม

```
┌─────────────────────┐     อ่านสด (anon key)    ┌──────────────────┐
│  GitHub Pages        │ ───────────────────────▶ │  Supabase Postgres│
│  (แดชบอร์ด static)   │                           │  customers/products/sales │
│  ลิงก์ถาวรฟรี        │ ◀─── fallback: data.js    └──────────────────┘
└─────────────────────┘
        ▲
        │ เปิดดู (มือถือ/คอม)
   ผู้บริหาร + พนักงาน
```

**ทำไมไม่ต้องมี server เอง:** แดชบอร์ดเป็น read-only → GitHub Pages เสิร์ฟไฟล์ static,
Supabase จัดการ Postgres ให้ → ไม่ต้องดูแลเซิร์ฟเวอร์ ไม่มีค่าใช้จ่ายขั้นต้น
